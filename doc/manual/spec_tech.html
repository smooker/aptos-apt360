This chapter describes those features of the processor or language with regard to motion commands that were sufficiently complex so as to warrant their exclusion from the basic discussion of motion commands (Section 5.4). It also includes extensions of concepts originally introduced in Chapter 5 in a simpler form.

14.1. Special Features

Certain options are available in the APT language that can override the normal manner in which the processor computes the required information for a series of motion commands. These options should be used when it is know that a special condition exists such that the normal assumptions made by the processor will not result in the desired output.

14.1.1. The Multiaxis Output (MULTAX) Statement

Many machine tools have the capability of controlling rotation in one or more directions, in addition to moving along the coordinate axes. The APT processor has the capability for outputting, in addition to the location of the end of the cutter, the components of the tool axis vector. This information is used later by the postprocessor to control the rotational axes on the machine tool. The statement that requests the APT processor to output the axis vector along with the end point of the tool is as follows:

The first form specifies that, from the location of the MULTAX statement on, all subsequent cutter locations will carry an associated axis vector for the rest of the part program. In certain instances this information is needed only in a portion of the part program. The second form is then available. The axis vectors will be output from the MULTAX/ ON statement until the MULTAX/ OFF statement is encountered. If both forms have been specified, the MULTAX statement takes precedence, and the axis vectors will be output for the rest of the part program.

Note

The individual postprocessor that is to be used at the conclusion of the cutter calculation phase may be set up only to recognize the MULTAX specification; that is, it may expect axis vectors for the rest of the part program as soon as any MULTAX specification is given. The user must be sure that the postprocessor will accept the alternate form (that is, MULTAX/ ON and MULTAX/ OFF) before it is specified in the part program.

14.1.2. The Processor Mode (2DCALC, 3DCALC, NDTEST) Statements

The APT processor generally uses an iterative technique to determine an acceptable cut vector in an elementary cut sequence such that the tolerances are not exceeded. However, the APT processor does have the capability of determining whether an analytical solution can be used, and it uses this technique whenever possible to provide a more efficient operation. The analytical method can be used under the following conditions:

For normal motion commands (that is, GOLFT/ S1, S2)

  • No multiple check surface specification

  • MULTAX not in effect

  • Vertical tool axis

  • DS and CS are vertical planes or cylinders

  • PSTAN not specified

  • PS is a non-vertical plane

  • No β angle specified in the definition of the cutter

For start-up statements (that is, GO/ S1, S2)

  • Maximum of two surfaces involved

  • MULTAX not in effect

  • THICK not specified for the PS

  • SFVCT not specified

  • INDIR not specified for the PS

  • No angle specified in the definition of the cutter

  • PS is a non-vertical plane

  • DS is a vertical plane or cylinder

The automatic selection of the analytical method by the processor can be overridden by the vocabulary word 3DCALC. When this word is specified on a card by itself, the processor uses the iterative method in all cases. This statement is modal; that is, it is in effect until overridden by another statement.

Two statements can override the 3DCALC statement. These are 2DCALC and NDTEST. The effect of these statements is identical -- the processor is to check again for the possible use of analytical calculations.

14.1.3. The Number of Points (NUMPTS) Statement

The iterative technique basically involves the following:

  • Compute a cut vector whose origin is the current cutter location.

  • Is it an acceptable vector (as long as possible, but still within tolerance)?

  • If not, go back and compute a new vector, and repeat until a satisfactory cut vector is obtained.

  • Each time a satisfactory vector is found, the processor investigates the check surface to see whether it has been reached. If it has, this motion command has been completed. If the CS has not been encountered, another cut vector is found on the basis of the end of the previous vector, and the whole operation is repeated.

This is a very rough outline of the way in which cut vectors -- and later, cut sequences -- are computed. Note, however, that this iterative technique can be stopped only when the check surface is reached. If an improper part program statement is specified, it is possible to generate cut vectors indefinitely along the same part surface and drive surface, never reaching the check surface. Figure 14.1 illustrates such a case.

Figure 14.1. Improper Check Surface

FROM/ PTA
GO/ DS
TLRGT, GORGT/ DS, CS

The processor avoids the possibility of an infinite loop by allowing only 400 cut vectors to be defined for any one elementary cut sequence. If the processor fails to encounter the check surface after computing 400 cut vectors along the same part and drive surface combination, an error is indicated.

It is possible that a particular situation may arise where more than 400 cut vectors would be required to reach a valid check surface. The NUMPTS statement provides the capability of overriding the 400 maximum set by the processor. The format of this statement is:

NUMPTS/ N

This statement specifies that the maximum number of cut vectors in any cut sequence shall not exceed N. The statement is modal; N will be in effect until a new NUMPTS statement is specified.

14.1.4. The Maximum Step (MAXDP) Statement

The rough outline given for the generation of a cut sequence actually contains two iterative schemes, one embedded in the other. As just discussed for NUMPTS, the one iteration involves computing more and more cut vectors until a check surface is reached. The other iteration is concerned with optimizing the length of the individual cut vector. The processor computes an acceptable cut vector that will not result in a violation of the specified tolerances. However, the processor does have an upper limit on the length of the individual vectors. This upper limit is taken to be ten units, unless otherwise specified in the part program. The upper limit can be controlled in the part program by the use of the MAXDP statement, whose format is as follows:

MAXDP/ A

The maximum length of the individual cut vectors is set to A units; this will continue in effect until a new MAXDP statement is specified. The following rules apply to the use of MAXDP:

  • The statement affects only the iterative scheme, not the analytical method.

  • This statement is ignored if both the drive and part surfaces are planes.

  • This statement affects only cut sequences resulting from a regular motion command, not those resulting from either point-to-point statements (GOTO, GODLTA) or start-up commands (GO, OFFSET).

14.1.5. The Thickness (THICK) Statement

The APT language permits a thickness to be added to any or all of the surfaces used in a motion or start-up statement. The format of the THICK statement is as follows:

THICK/ THPS, THDS, THCS, THCS1

This statement specifies that the thickness for the part surface is THPS, for the drive surface, THDS, and for the first and second check surfaces, THCS and THCS1, respectively. The processor assumes that the last thickness specified is to apply for the remaining surfaces; that is, THICK/.002, .004 results in the same processing as THICK/ .002, .004, .004, .004. The THICK statement is in effect for all surfaces until another THICK statement is specified.

A positive thickness specification results in a machined surface that will have excess stock left; a negative thickness specification results in an undercutting condition. An value may be specified for a positive thickness; however, because of the method used by the processor in implementing the THICK feature, a negative thickness must have a magnitude that does not exceed the cutter radius. For example, if the cutter radius is .5, any value T can be used for thickness as long as T ≥ -.5. Figure 14.2 and Figure 14.3 show examples of positive and negative specifications for THICK.

Figure 14.2. THICK, Add Material

FROM/ PTB
THICK/ 0, TH, 0
GO/ PAST, C1

FROM/ PTA
THICK/ 0, TH, 0
GO/ TO, C1

Figure 14.3. THICK, Remove Material

FROM/ PTB
THICK/ 0, -.25, 0
GO/ PAST, C1

FROM/ PTA
THICK/ 0, -.25, 0
GO/ C1
Note

THICK is ignored for any surface for which the cutter-to-surface relationship is ON. This is necessary because the processor is unable to determine what is meant by a positive or negative thickness when the end of the tool is on the surface in question.

14.1.6. The Delete Output (DNTCUT, CUT) Statements

The APT statement DNTCUT specifies that the processor is to execute the following commands involving cutter motion, but the resulting cutter points are not to be output. The CUT statement indicates that the processor is to resume outputting the cutter points. When a CUT statement is given, the current cutter location is output by the processor. This, in effect, causes a cutter motion directly from the last location preceding the DNTCUT statement to the current cutter location. This feature is used principally to solve start-up problems that require intermediate cutter positions without actually forcing the cutter to move through these intermediate positions.

Figure 14.4 illustrates a usage of CUT and DNTCUT. The desired start-up location is PTB. This can be reached by using a DNTCUT command and moving the cutter to PTC. A normal start-up command is then given, followed by a CUT statement. The resultant output points will be such that the cutter moves directly from PTA to PTB.

Figure 14.4. DNTCUT, CUT Usage

FROM/ PTA
DNTCUT
GO/ TO, PTC
GO/ S4, S2
CUT

When the DNTCUT mode is in effect, the points calculated, except for the final point, do not appear on the CLFILE and are therefore not available for debugging purpose. If it is desirable to see the intermediate points, DNTCUT/ PRINT may be specified. This will cause the intermediate points and their ISN'S (Internal Statement Numbers) to be output during ARELEM processing.

14.1.7. The Transfer statement (TRANTO) Statements

The concept of branching within a part program was discussed in Chapter 8. All of the branching operations introduced there are actually executed by the processor before the time the cutter paths are computed. There are instances when the desired branching can be acted upon only on the basis of the results of executing a statement involving movement of the tool. This can be done by using the TRANTO statement. This statement will be ignored by the other portions of the processor and will be acted upon only when the cutter paths are computed. The format of this statement is as follows:

TRANTO/ ID1

Figure 14.5. TRANTO Usage

When the arithmetic element (the portion of the APT processor that computes the cutter points) encounters this statement, control is passed immediately to the statement whose label is ID1. The statement labeled ID1 must generate a PROTAP record. (For contents of PROTAP, see section titled "Postprocessor Interface" in IBM manual N/C 360 APT Application Description (H20-0181).) This capability is useful in certain multiple check surface cases (see Section 5.4.3.4). An example is shown in Figure 14.5. The same contour is to be machined with two different-size cutters (a rough and finish cut). The fillet defined as S2 requires that multiple check surface branching be used. In addition, the motion along S4 requires a GOFWD for the small cutter and GORGT for the large cutter. The following set of APT statements would be required for the large cutter: 

FROM/ PTA
GO/ S1, PS, ON, S3
TLRGT, GORGT/ S1, TO, S4
GORGT/ S4
GOLFT/ S5
$$
$$ Other APT Statements
$$

However, the following instructions would be required for the small cutter: 

FROM/ PTA
GO/S1, PS, ON, S3
TLRGT, GORGT/ S1, TANTO, S2
GOFWD/ S2, TANTO, S4
GOFWD/ S4, PAST, S5
GOLFT/ S5
$$
$$ Other APT Statements
$$

Using the concept of multiple check surfaces and the TRANTO statement, the two sets of instructions can be combined as follows: 

FROM/ PTA
GO/ S1, PS, ON, S3
TLRGHT, GORGHT/ S1, TANTO, S2, ID1, TO, S4, $
ID2
ID1) GOFWD/ S2, TANTO, S4
GOFWD/ S4, PAST, S5
TRANTO/ ID3
ID2) GORGT/ S4
ID3) GOLFT/ S5
$$

14.1.8. The Special Cutter Option

This statement defines a disk or ring cutter, with radius equal to RA and height along the tool axis from the bottom of the cutter equal to HI. The selection of the integer 1 or 2 indicates:

  1. The cutter is to be used for the part surface.

  2. The cutter is to be used for the drive surface.

The cutter specified by this statement is used instead of the real cutter for the indicated surface.
